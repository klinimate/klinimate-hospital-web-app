# Klinimate Brand Guidelines

Version: 2.0

Status: Active

Owner: Dr. Avdhut Kulkarni (Founder & Director)

---

# Purpose

This document defines the official brand identity, visual language, product naming conventions, communication style, and design standards for the Klinimate ecosystem.

Every Klinimate product—including web applications, mobile applications, AI services, command centre, virtual specialist network, websites, APIs, documentation, exported clinical reports, and marketing material—must follow these standards.

This document serves as the single source of truth for all Klinimate branding.

---

# Brand Identity

Klinimate is an AI-Powered Clinical Intelligence Platform.

It empowers healthcare professionals to deliver safer, smarter, standardized, and specialist-supported patient care across every healthcare setting.

The brand should communicate:

- Trust
- Clinical Excellence
- Intelligence
- Simplicity
- Reliability
- Innovation
- Global Healthcare

Every interaction should feel calm, professional, modern, and effortless.

---

# Vision

To build the world's most trusted AI-Powered Clinical Intelligence Platform that empowers healthcare professionals to deliver safer, smarter, standardized, and specialist-supported patient care across every healthcare setting.

# Official Product Names

Always use:

- Klinimate Platform
- Klinimate Intelligence Engine
- Patient Intelligence
- Klinimate Care Coordinator
- Klinimate Virtual Specialist Network
- Klinimate Command Centre
- Klinimate Admin
- Healthcare Organization Workspace

Avoid user-facing abbreviations such as:

- KIE
- KCC
- KVSN
- AI-CDSS

Abbreviations may be used internally by developers but should not appear in the user interface or customer-facing documentation.

---

# Naming Philosophy

Use **Klinimate** when referring to the platform.

Examples:

- Klinimate Platform
- Klinimate Mobile App
- Klinimate Website
- Klinimate Command Centre

Use **Patient Intelligence** when referring to the clinician-facing AI summary.

Examples:

- Review Patient Intelligence.
- Finalize Patient Intelligence.
- Generate Patient Intelligence Summary.

Use **Klinimate Intelligence Engine** when referring to the backend AI engine.

Examples:

- Klinimate Intelligence Engine analyzed the uploaded documents.
- Klinimate Intelligence Engine generated Patient Intelligence.

---

# Clinical Colour System

Clinical colours communicate patient status.

They must never be used as branding colours.

- 🟢 Green — Stable
- 🟡 Amber — Needs Review
- 🔴 Red — Urgent / Critical
- ⚫ Black — Black Triage / Deceased

Brand colours communicate identity.

Clinical colours communicate clinical status.

The two colour systems must remain separate.

# User Experience Philosophy

Klinimate should feel like:

**WhatsApp + ChatGPT for Healthcare**

The experience should be:

- Mobile-first
- AI-first
- Clinician-first
- Fast
- Simple
- Professional
- Human-centred

Patient information should always take priority over branding.

---

# User Interface Principles

Every screen should:

- Use a white background.
- Prioritize patient information.
- Minimize scrolling.
- Reduce typing.
- Support one-handed mobile use.
- Present one primary action.

Follow:

Click More • Type Less

Search • Select • Save

# Cards

- White background
- Rounded corners
- Soft shadow

- Minimal borders
